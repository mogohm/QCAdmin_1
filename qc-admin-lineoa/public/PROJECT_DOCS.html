Duplicate prevention7-day window check ป้องกัน message เดิมถูก insert ซ้ำจากการ scrape ซ้ำ✅ พร้อม

Auto-detect สมัครตรวจเบอร์โทร 0[689]\d{8} หรือเลขบัญชี 10–12 หลัก จาก 5 customer messages ล่าสุด✅ พร้อม

ฟีเจอร์ที่ขาด (สำคัญ)

  • 🔴
    Knowledge Rules UI — API GET/POST /api/config/rules พร้อมแล้วแต่ไม่มีหน้าจัดการ ตอนนี้ต้องเรียก API ตรง หรือแก้ใน sql/schema.sql ทำให้ correctness score ใช้ heuristic เป็นหลัก ซึ่งไม่แม่นยำเท่า rules จริง
  • 🔴
    Conversation Close / Reopen UI — DB มี conversations.status แต่ไม่มีปุ่ม toggle ใน frontend Open Cases จึงสะสมไม่มีวันลด
  • 🟠
    Customer Profile Page — ไม่มีหน้าดูประวัติลูกค้าทั้งหมด (บทสนทนาทั้งหมด, events, total deposits)
  • 🟠
    Export / Report — ไม่มี export CSV หรือ print-friendly PDF สำหรับ ranking/reply log

ข้อจำกัดทางเทคนิค

  • 🟡
    KYC Auto-detect — ระบบ detect register (phone/bank) และ deposit อัตโนมัติ แต่ KYC ต้องยิง POST /api/customer-event จากระบบหลังบ้านเอง
  • 🟡
    LINE Webhook รับแค่ text — Sticker, รูปภาพ, ไฟล์, Location, Flex Message ไม่ถูกบันทึก ทำให้ conversation context ขาดช่องว่างสำหรับ message ที่ไม่ใช่ text
  • 🟡
    Data เก่าก่อน timestamp fix — Row ที่ insert ก่อนระบบ timestamp จะมี customer_message_id = NULL ทำให้คอลัมน์ คำถาม ใน reply log แสดง "—" สำหรับข้อมูลเก่า (ข้อมูลใหม่ทุกอย่างถูกต้อง)
  • 🟡
    Auth แบบ simple — ป้องกันด้วย x-api-key header เท่านั้น ไม่มีระบบ login/logout/session management จริงสำหรับหลายผู้ใช้
  • ⚪
    Single LINE Account — Scraper ผูกกับ auth.json ไฟล์เดียว รองรับ 1 LINE OA account ถ้ามีหลาย OA ต้องรัน scraper instance แยก
  • ⚪
    No Real-time Push — Dashboard refresh ทุก 30 วินาที ยังไม่มี WebSocket หรือ SSE เมื่อลูกค้าส่งข้อความเข้ามา ต้องรอรอบถัดไปถึงจะเห็น

Command Line Arguments

# Login และบันทึก session ก่อน (ทำครั้งแรก หรือเมื่อ session หมดอายุ)
node login.js

# Scrape ครั้งเดียว (ไม่ loop)
node scraper.js

# Watch mode — poll ทุก 10 วินาทีตลอด
node scraper.js --watch

# Auto-create job ทุก 30 นาที + watch
node scraper.js --schedule=30

# แสดง browser window (debug)
node scraper.js --headed --watch

# หรือตั้งผ่าน .env

ที่ตั้ง: lib/qc-engine.js — เรียกใช้ใน /api/admin/log-reply และ /api/admin/reply

สูตรคะแนน

30%
100 ถ้าตอบภายใน SLA
ลด 5 คะแนนทุก 30 วินาที

40%
Rule-based (knowledge_rules)
หรือ heuristic topic match

30%
Polite words +7
Empathy words +8
Rude words −30
Final Score
= Σ
Bot penalty −15 ถ้า
มี 2+ bot phrases

เกณฑ์ผ่าน/ไม่ผ่าน

ระดับScoreสีป้าย
ดี≥ 85●✅ ดี
พอใช้70–84●⚠️ พอใช้
ต่ำ< 70●❌ ต่ำ + Telegram alert

Fail Reasons

"ตอบช้ากว่า SLA"           → speed_score < 80
"คำตอบอาจไม่ตรงประเด็น..."  → correctness_score < 70
"น้ำเสียงหรือ service mind ต่ำ" → sentiment_score < 70
"มี pattern ตอบเหมือนบอท"  → bot_penalty > 0

API Reference

Authentication: ส่ง header x-api-key: YOUR_ADMIN_API_KEY กับทุก endpoint ที่ต้องการสิทธิ์

Dashboard

MethodPathDescription
GET /api/dashboard ดึงข้อมูลทั้งหมด · params: ?from=YYYY-MM-DD&to=YYYY-MM-DD
Response: { kpi, ranking, weeklySummary, promos, openCases, replyLog, lastActivity }

Admin

MethodPathDescription
POST /api/admin/log-reply รับข้อมูลจาก Scraper · body: { line_user_id, text, admin_name, customer_text?, admin_ts?, customer_ts? }
Response: { ok, qc: { finalScore, speedScore, ... } }
POST /api/admin/reply ส่ง reply ผ่านระบบ (push LINE + บันทึก QC) · body: { conversation_id, admin_id, text, send_line? }
POST /api/admin/import Import admin จาก LINE OA Manager · body: { text: "PK Admin1\nPK Admin2" }
GET /api/admin/list รายชื่อ admin ทั้งหมดที่ is_active = true

Scraper Job Queue

MethodPathDescription
POST /api/scraper/job สร้าง job ใหม่ · body: { date_from, date_to }
GET /api/scraper/job ดูสถานะ job ปัจจุบัน (running/pending)
GET /api/scraper/poll Scraper ดึง job ที่ pending อยู่ (1 job)
PATCH /api/scraper/poll Scraper อัพเดทสถานะ · body: { id, status?, logged_count?, current_chat?, error_text? }
POST /api/scraper/init สร้างตาราง scraper_jobs (รัน 1 ครั้งแรก)

Webhook & Events

MethodPathDescription
POST /api/webhook LINE webhook endpoint · verify HMAC-SHA256 signature · บันทึก customer message
POST /api/customer-event บันทึก event จากระบบหลัง · body: { line_user_id, event_type, status?, amount?, promotion_code? }
event_type: register | kyc | deposit | promotion
GET /api/config/rules ดู Knowledge Rules ทั้งหมด
POST /api/config/rules เพิ่ม/แก้ rule · body: { rule_code, rule_name, category, question_keywords[], answer_keywords[], weight }
GET /api/health ตรวจสถานะ DB + LINE token + ENV + 5 messages ล่าสุด (ไม่ต้องใช้ key)

ขั้นตอนติดตั้ง (First Time)

1

สร้าง Neon PostgreSQL

เข้า neon.tech สร้าง project ใหม่ คัดลอก DATABASE_URL ไว้

รัน SQL schema: เปิด Neon SQL Editor แล้ว paste เนื้อหาจาก sql/schema.sql

2

ตั้งค่า LINE Messaging API

เข้า LINE Developers Console → เลือก Channel → Messaging API

คัดลอก Channel secret และ Channel access token (long-lived)

ตั้ง Webhook URL: https://YOUR_DOMAIN/api/webhook → กด Verify

3

Deploy บน Vercel

Push โฟลเดอร์ qc-admin-lineoa/ ขึ้น GitHub แล้ว import ใน Vercel

ตั้ง Environment Variables ทั้งหมดในตาราง "Next.js App" ด้านบน

หลัง deploy สำเร็จ เรียก POST /api/scraper/init เพื่อสร้างตาราง scraper_jobs

4

Import Admin

เปิดหน้า /admin → กรอก API Key → เข้า LINE OA Manager → Settings → Manage permissions

คัดลอกชื่อ Member ที่มี prefix PK มาวางในช่อง Import

5

Setup Playwright Scraper

cd qc-scraper
npm install
# สร้าง .env ตามตัวอย่างด้านบน
node login.js       # login ด้วย browser window จริง
node scraper.js --watch --schedule=30  # เริ่ม scrape
6

(Optional) ตั้งค่า Telegram Alert

สร้าง bot ผ่าน @BotFather → รับ token → เพิ่ม bot เข้า group/channel → รับ chat_id

ตั้ง TELEGRAM_BOT_TOKEN และ TELEGRAM_CHAT_ID ใน Vercel ENV

⚠️ Session หมดอายุ: LINE OA login session ใน auth.json จะหมดอายุเป็นระยะ เมื่อ scraper แจ้ง error "Session หมดอายุ" ให้รัน node login.js อีกครั้ง
💡 เช็คสถานะระบบ: เปิด /admin แล้วกด "เช็คระบบ" เพื่อตรวจสอบ DB connection, LINE token, และ recent messages ทั้งหมดในที่เดียว
